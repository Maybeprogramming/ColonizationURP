# Flower Geometry Shader

Процедурный шейдер цветов с геометрическим шейдером для URP. Генерирует стебель и три типа цветов (ромашка, тюльпан, мак) из одной точки, с процедурной рандомизацией формы.

## Использование в проекте

### 1. Подготовка

1. Создайте или выберите **Material** в папке `Resources/Flawers/`.
2. В инспекторе материала выберите шейдер `Custom/FlowerGeometry`.
3. Создайте на сцене пустой `GameObject` и добавьте компонент `FlowerRenderer`.

### 2. Настройка FlowerRenderer

| Параметр | Описание |
|---|---|
| `Material` | Созданный материал с шейдером `Custom/FlowerGeometry` |
| `Positions` | Массив `Vector4` — координаты цветка (`xyz`) + seed (`w`) для рандомизации |

Альтернативно используйте компонент `FlowerSpawner`, который автоматически заполняет `Positions` по рейкасту на terrain.

## Параметры материала (Inspector)

### Stem (Стебель)

| Параметр | Описание |
|---|---|
| `Stem Height` | Высота стебля в мировых единицах (0.3–3) |
| `Stem Width` | Ширина стебля (0.01–0.15) |
| `Stem Bottom Color` | Цвет основания стебля |
| `Stem Top Color` | Цвет верхушки стебля |

### Daisy (Ромашка)

| Параметр | Описание |
|---|---|
| `Daisy Petal Color` | Цвет лепестков ромашки |
| `Daisy Center Color` | Цвет сердцевины ромашки |

### Tulip (Тюльпан)

| Параметр | Описание |
|---|---|
| `Tulip Color` | Цвет тюльпана (лепестки цельнокроеные) |

### Poppy (Мак)

| Параметр | Описание |
|---|---|
| `Poppy Color` | Цвет лепестков мака |
| `Poppy Center Color` | Цвет сердцевины мака |

### Global (Общие)

| Параметр | Описание |
|---|---|
| `Head Size` | Размер головки цветка (0.1–1.5). Влияет на длину/ширину лепестков и сердцевины |
| `Wind Speed` | Скорость ветра — частота колебаний стебля |
| `Wind Strength` | Сила ветра — амплитуда отклонения |
| `Height Jitter` | Разброс высоты стеблей (0 — все одинаковые, 1 — максимальная вариация) |

## Алгоритм работы

### Vertex Shader (`vert`)
Передаёт `SV_InstanceID` в геометрический шейдер. Вершины не обрабатываются.

### Geometry Shader (`geom`)
Для каждого экземпляра (`maxvertexcount = 72`):

1. **Стебель** (12 вершин):
   - Два billboard-квада (поперечный и продольный), как у травы.
   - Верхушка стебля сужается (0.3 от ширины основания).
   - Цвет интерполируется от `_StemColorBottom` к `_StemColorTop`.
   - Ветер отклоняет верхнюю часть стебля.

2. **Тип цветка** — определяется по `seed`:
   - `typeRand * 3` → 0: ромашка, 1: тюльпан, 2: мак.

#### Ромашка (Daisy)
- **8 лепестков** по кругу, каждый — отдельный квад (6 вершин).
- Лепесток: узкий у основания (`0.035 * headSize`), расширяется к концу.
- Сердцевина: 2 billboard-квада, жёлтого цвета.

#### Тюльпан (Tulip)
- **5 лепестков** по кругу.
- Лепестки широкие (`0.35 * headSize`), образуют закрытую чашу.
- Без отдельной сердцевины.

#### Мак (Poppy)
- **4 крупных лепестка** по кругу.
- Лепестки очень широкие (`0.5 * headSize`), слегка опущены вниз.
- Тёмная сердцевина из 2 billboard-квадов.

3. **Рандомизация**:
   - `GetRandom(seed + 1)` — высота стебля.
   - `GetRandom(seed + 2)` — ширина стебля.
   - `GetRandom(seed + 3)` — размер головки.
   - `GetRandom(seed + 5)` — тип цветка.

### Fragment Shader (`frag`)
- Цвет из вершинных атрибутов (передаётся из геометрии).
- То же полу-Lambert освещение, что и у травы.
- Shadow attenuation от основного источника света.
- Ambient-добавка `+ 0.3`.

### Shadow Caster Pass
Полный дубль геометрии с `ApplyShadowBias` для всех элементов (стебель + лепестки + сердцевина).

## Использованные технологии

- **Geometry Shader** — процедурная генерация до 72 вершин на точку.
- **GPU Instancing** через `StructuredBuffer<float4>` + `SV_InstanceID`.
- **Billboard-ориентация** для стебля и сердцевины.
- **Procedural pseudo-random** для вариативности типа, размера и формы.
- **URP Forward + ShadowCaster** — два прохода для корректного освещения и теней.
- **Вспомогательная функция `EmitQuad`** — универсальная эмиссия квада из 4 вершин (упрощает код, хотя в шейдере используется и прямой вызов).
- **Pi (`6.28319`)** — для кругового расположения лепестков.
