# Grass Geometry Shader

Процедурный шейдер травы с геометрическим шейдером для URP. Генерирует двусторонние billboard-квады травы из точек, создавая плотный травяной покров с минимальным расходом памяти.

## Использование в проекте

### 1. Подготовка

1. Создайте или выберите **Material** в папке `Resources/Grass/`.
2. В инспекторе материала выберите шейдер `Custom/GrassGeometry`.
3. Создайте на сцене пустой `GameObject` и добавьте компонент `GrassRenderer`.

### 2. Настройка GrassRenderer

| Параметр | Описание |
|---|---|
| `Material` | Созданный материал с шейдером `Custom/GrassGeometry` |
| `Positions` | Массив `Vector4` — координаты травы (`xyz`) + seed (`w`) для рандомизации |

Альтернативно используйте компонент `GrassPainter`, который автоматически заполняет `Positions` по рейкасту на terrain.

## Параметры материала (Inspector)

### Blade Height
`Range(0.1, 2)` — **Высота травинки** в мировых единицах. Значение умножается на jitter для разнообразия.

### Blade Width
`Range(0.01, 0.2)` — **Ширина травинки**. Узкие для тонкой травы, широкие для крупной.

### Wind Speed
`Range(0, 5)` — **Скорость ветра**. Частота синусоидальных колебаний.

### Wind Strength
`Range(0, 1)` — **Сила ветра**. Амплитуда отклонения травинок.

### Height Jitter
`Range(0, 1)` — **Разброс высоты**. При 0 все травинки одинаковой высоты, при 1 разброс максимальный.

### Base Color
`Color` — **Цвет основания** травинки. Интерполируется к Top Color по UV.y.

### Top Color
`Color` — **Цвет верхушки** травинки.

## Алгоритм работы

### Vertex Shader (`vert`)
Передаёт `SV_InstanceID` в геометрический шейдер. Сами вершины не обрабатываются — вся геометрия строится процедурно.

### Geometry Shader (`geom`)
Для каждого экземпляра (точки из буфера `_Positions`):

1. **Считывание данных**: позиция `root` (xyz) и `seed` (w).
2. **Рандомизация**: функция `GetRandom(seed)` генерирует псевдослучайные числа для разброса высоты (`_Jitter`), ширины (0.8–1.2 от `_BladeWidth`) и фазы ветра.
3. **Расчёт ветра**: `sin(time * _WindSpeed + seed * 2π) * _WindStrength` — синусоидальное отклонение в направлении `bladeRight`.
4. **Billboard**: направление `bladeRight` = cross(up, viewDir). Квады всегда обращены к камере.
5. **Генерация двух квадов** (по 6 вершин каждый, `maxvertexcount = 12`):
   - Квад 1: ориентация по `bladeRight` — поперечный разрез.
   - Квад 2: ориентация по `viewDir` — продольный разрез.
   - Каждый квад: две вершины у корня, одна на верхушке + дополнительная вершина на верхушке с небольшим расширением для формы.
6. **UV-координаты**: `(0,0)`, `(1,0)`, `(0.5,1)` — для градиента цвета по высоте.

### Fragment Shader (`frag`)
- Интерполяция цвета: `lerp(_BaseColor, _TopColor, uv.y)`.
- Простейшее освещение: `NdotL` с полу-Lambert (`* 0.5 + 0.5`), учёт тени от основного источника света.
- Ambient-добавка `+ 0.3` для освещения теневых сторон.

### Shadow Caster Pass
Полный дубль геометрического шейдера с `ApplyShadowBias` для корректного отбрасывания теней.

## Использованные технологии

- **Geometry Shader** (`#pragma require geometry`, target 4.6) — процедурная генерация треугольников на GPU.
- **GPU Instancing** через `SV_InstanceID` + `StructuredBuffer<float4>` — тысячи травинок за один draw call.
- **Billboard-ориентация** — поворот геометрии к камере без поворота mesh-объекта.
- **Procedural pseudo-random** — `frac(sin(seed * C1) * C2)` для вариативности без CPU-оверхэда.
- **URP Lighting** — `Lighting.hlsl`, `GetMainLight()`, shadow attenuation.
- **Two-pass rendering** — Forward + ShadowCaster для корректных теней.
